Singapore Mobility Decision View

Context

In dense cities, users do not only need transport data. They need a practical decision: book now, walk to a better pickup point, switch to public transport, or wait.

Product hypothesis

Public taxi, transit, road, and disruption signals can be combined into a lightweight, explainable decision layer for mobility choices.

Key decisions

  • Use public mobility data only.
  • Avoid claims about private platform pricing, demand, or driver allocation.
  • Keep API keys server-side.
  • Avoid persistent user-location storage.
  • Use explainable proxy scoring instead of black-box prediction.

Architecture

User location
  ↓
Frontend map UI
  ↓
FastAPI backend
  ↓
LTA DataMall APIs
  ↓
Taxi / road / transit signals
  ↓
Scoring layer
  ↓
Pickup readiness + fallback choice

Singapore Port Traffic Tracker

Context

Raw vessel movement signals are difficult to interpret without a clear operational map, simple categories, and live status indicators.

Product hypothesis

Live AIS vessel signals can be transformed into a lightweight visibility product for vessel movement, vessel categories, and traffic awareness.

Key decisions

  • Use live AIS streaming data.
  • Separate backend data ingestion from frontend map rendering.
  • Expose signal status so users understand whether live data is flowing.
  • Present vessel type and traffic metrics in a simple interface.

Architecture

AISStream WebSocket
  ↓
FastAPI backend
  ↓
Vessel cache + status cache
  ↓
/ships + /metrics APIs
  ↓
Mapbox frontend
  ↓
Live vessel map + traffic indicators
